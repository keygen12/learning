#ПОДКЛЮЧЕНИЕ К СЕРВЕРУ
Чтобы подключиться к серверу мы должны ввести `ssh имя_пользователя@название_сервера` или же `ssh имя_пользователя@IP-адрес_сервера` - выглядит так (`ssh keygen@192.168.0.34`) 
или `ssh keygen@server1`

#СОЗДАНИЕ КЛЮЧЕЙ ДЛЯ АВТОМАТИЧЕСКОГО СОЕДИНЕНИЯ С СЕРВЕРОМ, БЕЗ ИСПОЛЬЗОВАНИЯ ПАРОЛЯ
Чтобы создать ключ мы должны ввести команду `ssh-keygen -t ed25519 -f ~/.ssh/backend-server`
`ssh-keygen` - генерирует Публичный и Приватный ключи ssh
`-t ed25519` - ключ в формате шифрования ed25519 
`-f ~/.ssh/backend-server` - сохраняем ключ в папку .ssh и даем название ключу backend-server

#Добавление ключа на сервер 
Чтобы добавить ключ на сервер нужно ввести команду `ssh-copy-id имя_пользователя@название_сервера`
Что делает команда `ssh-copy-id`
🔧 Команда
`ssh-copy-id user@server`
⚙️ Что происходит внутри (по сути те же команды)
1. Берётся твой публичный ключ
`cat ~/.ssh/id_rsa.pub`

(или id_ed25519.pub)

2. Подключение к серверу по SSH (с паролем)
`ssh user@server`
3. Создаётся папка .ssh, если её нет
`mkdir -p ~/.ssh`
4. Выставляются правильные права
`chmod 700 ~/.ssh`
5. Ключ добавляется в authorized_keys
`echo "ТВОЙ_КЛЮЧ" >> ~/.ssh/authorized_keys`
6. Права для файла с ключами
`chmod 600 ~/.ssh/authorized_keys`
🧠 Вся команда одной строкой (почти как ssh-copy-id)

Вот эквивалент вручную:
`cat ~/.ssh/id_rsa.pub | ssh user@server "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"`
✅ Итог
твой ключ оказался на сервере
SSH теперь сравнивает ключи вместо пароля
вход становится без пароля

#ЕСЛИ СЕРВЕРОВ НЕСКОЛЬКО И КЛЮЧЕЙ ТОЖЕ?
Тогда применяем `-i ~/.ssh/название_приватного_ключа`
Полная команда `ssh -i ~/.ssh/название_приватного_ключа@название_сервера`	Пример `ssh -i ~/.ssh/backend-server keygen@192.168.0.34`

#НАСТРОЙКА ФАЙЛА config
Чтобы каждый раз не писать название сервера и расположение ключа к нему, нужно просто настроить файл config 
Он лежит в `~/.ssh/config`
Пример ```
Host server1
    HostName 10.0.0.1
    User root
    IdentityFile ~/.ssh/id_rsa

Host server2
    HostName example.com
    User admin
    Port 2222
    IdentityFile ~/.ssh/id_ed25519
```
Объяснение:
🔹 `HostName 192.168.1.10`

Реальный адрес сервера (IP или домен)

🔹 `User user`

Имя пользователя на сервере
👉 Не нужно писать user@... каждый раз

🔹 `Port 22`

Порт SSH (если стандартный — можно даже не указывать)

🔹 `IdentityFile ~/.ssh/id_ed25519`

Какой ключ использовать для подключения


И теперь вместо этого: `ssh -i ~/.ssh/id_ed25519 user@192.168.1.10 -p 22`
Ты пишешь это: `ssh server1` или `ssh server2`
